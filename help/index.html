First 10 seconds

Choose the guide that matches how you use EnerZee.

The app changes what it shows based on intent. The help center follows the same rule so training-first users are not pushed into unrelated tracking.

Task guides

Do the thing you came for.

Each card gives the exact app path, what to verify, and when to report a problem instead of retrying blindly.

Use Today

Today is the first tab and shows the most useful actions for the day.

  1. Open Today from the first tab.
  2. Start with the top focus card, then use the visible task cards for the day.
  3. Use the Help icon at the top if a card looks wrong or a task is missing.
Report if Today shows stale cards or actions that do not open.

Use an active workout

Active Workout keeps the session running while you log sets, rest, or minimize.

  1. Open Workout and start a program or custom workout.
  2. Type today's kg and reps into the current set rows.
  3. Use rest banner controls during rest; minimizing keeps the session alive.
  4. If you leave the screen, return from the active workout banner before starting over.
Report if typed sets disappear, the app crashes, or old history appears as today's logged data.

Find or add an exercise

Exercise search should help you start or edit sessions without blocking your workout.

  1. Open Workout and use add or swap exercise from the workout flow.
  2. Search the closest exercise name first.
  3. If the exercise is missing, use the custom exercise path or report the exact name from Help.
Report the missing exercise name if search cannot find or save it.

Use Progress

Progress shows trends, tappable insights, and recent proof from workouts, body metrics, habits, and devices.

  1. Open Progress from the bottom tab.
  2. Choose the relevant section, then tap cards that show a chevron or clear action styling.
  3. If a completed workout is missing, refresh the app and check the date filter.
Report if a card looks tappable but does nothing, or if a workout does not update Progress.

Use Food and macro views

For users who enabled food support, Food and Progress nutrition views show logs, calories, and macros.

  1. Open Food to log or review items for the selected day.
  2. Open Progress and use Nutrition to review calorie and macro trends.
  3. If totals look wrong, check the selected day and logged item portions.
Report if totals differ after refresh or the day selection is wrong.

Check connected devices

Wearable settings show connection and permission state. EnerZee should not claim imported provider data unless the app shows it.

  1. Open Profile, then connected devices or wearable settings.
  2. Check whether the provider shows connected, disconnected, or needs permission.
  3. If Fitbit returns to sign-in or Apple Health does not connect, report the provider and what happened after the tap.
Visible in the current app: Fitbit, WHOOP, and Apple Health on iOS; Health Connect appears on Android.

Fix account, billing, or privacy requests

Account access issues are usually specific to sign-in, verification, subscription, or device state.

  1. Check that the same email or provider was used to sign in.
  2. Close and reopen EnerZee so the app can refresh access.
  3. For export, correction, or deletion requests, use the email tied to your EnerZee account.
Use support for purchase account, device, locked screen, or privacy request details.

Check EnerZee status

Status checks explain the likely failure area without claiming a live provider outage.

  1. Open Help and review Status checks.
  2. Retry the failed action once if the app is responsive.
  3. For provider handoff issues, check permissions and whether the provider returned to EnerZee.
Report if the same action fails twice or the app cannot show a current state.
AI Coach boundaries

What Coach can and cannot do for app help.

Coach can explain supported app tasks. It should not invent controls, claim sync success without visible app state, or pretend a report was reviewed.

Coach can help with

  • Finding the right screen or tab.
  • Explaining Today, Workout, Progress, Food, Profile, and Help actions.
  • Turning unclear app issues into a useful bug report.
  • Explaining what evidence the app currently shows.

Coach should never do

  • Invent a button, page, setting, or integration that is not in the approved guide.
  • Claim Apple, Fitbit, WHOOP, or Health Connect data synced unless the app shows it.
  • Say a bug is resolved before the app or support team confirms it.
  • Push unrelated tracking to training-first users.
Support path

Send a report that the team can actually use.

Best report: what screen you were on, what you tapped, what you expected, what happened, app build, device, and screenshot. If the app is open, use the Help icon from the screen where the issue happened.
